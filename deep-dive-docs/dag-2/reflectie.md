# Reflectie

## Mermaid

Q: Waar helpt een diagram je bij?
A: van te vorgen plannen hoe een systeem geboud moet worden, en een visueel geven van het het systeem.

Q: Wat is onduidelijk zonder diagram?
A: zonder diagram zou het onduidelijk zijn om hoe een bepaaled systeem hoort te werken en communiceren, wat in een team werken best moeilijk zou maken.

## Cursor

Q: Wat snapte je eerst niet?
A: Ik snapte niet echt op welke manier cursor perse beter zou zijn dan het gebruiken van iets zoals ChatGPT. Verder wist ik niet of het makkelijk was om mee te werken was.

Q: Wat snap je nu wel?
A: Ik snap nu dat het gebruiken van een AI IDE (Cursor) best makkelijk is (ook omdat het een fork is van VS code), en hoe je er mee code kan laten schijven en reviewen.

Q: Zou je deze code zelf durven aanpassen?
A: Met wat cursor had uitgelegt over de gegenereerde code zou ik wel durgen wat veranderingen te maken. al weet ik niet zeker of dit hetzelfde zou zijn als het gegenereede code meer dan een paar files zou zijn.

## Wanneer is een CLI-tool beter dan IDE of browser?

voor wanneer je wil dat iets makkelijk bij alle files op jouw computer kan voor een analysis.

## Top drie fav tools

- mermaid
ik vind uml gebruiken niet fijn, dus het is wel fijn dat het dit voor me kan doen.
- minstral-cli
handig voor redelijk snelle code analysis, best hoge gratis token count.
- cursor
kan heel veel, en is redelijk snel, maar ik schrijf liever mijn code zelf.
